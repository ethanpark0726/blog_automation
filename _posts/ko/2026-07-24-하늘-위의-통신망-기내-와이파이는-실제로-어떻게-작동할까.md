# 하늘 위의 통신망: 기내 와이파이는 실제로 어떻게 작동할까?

지상 35,000피트 상공에서 시속 800km가 넘는 속도로 비행하며 스마트폰으로 SNS를 확인하거나 이메일을 보낼 때, 문득 이런 궁금증이 생긴 적 없으신가요? "도대체 이 높은 하늘 위에서 내 스마트폰은 어떻게 인터넷에 연결되는 걸까?" 이는 마법처럼 느껴질 수 있지만, 사실 고도의 항공 우주 공학이 집약된 복잡한 기술의 산물입니다. 기내 와이파이(In-Flight Wi-Fi, IFW)의 원리를 이해하려면, 기내에서 이루어지는 로컬 연결과 항공기와 전 세계 인터넷을 연결하는 장거리 '백홀(Backhaul)' 연결을 구분해서 살펴봐야 합니다.

## 로컬 연결: 내 기기와 기내 네트워크의 만남

스마트폰에서 '기내 와이파이' 네트워크에 접속할 때, 여러분의 기기는 위성이나 지상 기지국에 직접 연결되는 것이 아닙니다. 대신, 기내에 설치된 **온보드 액세스 포인트(Onboard Access Point, OAP)**에 연결되는 것입니다.

상업용 항공기 내부는 가정이나 사무실과 마찬가지로 국소적인 와이파이 네트워크 환경을 갖추고 있습니다. 기내 곳곳에는 여러 개의 무선 액세스 포인트(WAP)가 분산 배치되어 있으며, 이들은 이더넷 케이블을 통해 중앙의 **온보드 서버(Onboard Server)**와 연결됩니다. 이 서버는 교통 관제사 역할을 수행하며 대역폭 관리, 콘텐츠 필터링, 그리고 캡티브 포털(Captive Portal) 로그인 과정을 처리합니다. 2004년부터 많은 항공사가 이러한 접속 방식을 기내 엔터테인먼트 서비스와 통합해 왔습니다.

### 온보드 네트워크의 기술 스택
기내 네트워크는 일반적으로 표준 802.11 프로토콜을 사용합니다. 여러분의 기기 입장에서 이 연결은 로컬 네트워크일 뿐입니다. 진정한 '마법'은 서버 뒤에서 일어납니다. 서버는 수백 명의 승객이 사용하는 데이터를 하나로 모아 기체 밖으로 전송하기 위한 단일 데이터 스트림으로 통합합니다.

```mermaid
graph LR
    "User Device" -- "Wi-Fi 802.11" --> "Cabin Access Point"
    "Cabin Access Point" -- "Ethernet" --> "Onboard Server"
    "Onboard Server" -- "Satellite/ATG" --> "Ground Station/Satellite"
    "Ground Station/Satellite" --> "The Internet"
```

## 백홀: 항공기가 세상과 소통하는 방법

데이터가 온보드 서버에 도달하면, 이제 지상으로 전송되어야 합니다. 이를 위한 주요 방법은 두 가지가 있습니다. 바로 **지상 통신(Air-to-Ground, ATG)**과 **위성 통신(Satellite, SATCOM)**입니다.

### 1. 지상 통신 (Air-to-Ground, ATG)
ATG는 여러분의 휴대전화가 작동하는 방식과 유사합니다. 항공기 하단에 장착된 안테나가 지상의 기지국 네트워크와 통신합니다. 비행기가 이동함에 따라 연결은 하나의 기지국에서 다음 기지국으로 자연스럽게 넘어갑니다(핸드오프). 과거 Gogo의 ATG-4와 같은 시스템은 여러 항공사에 상당한 속도 향상을 제공했습니다.
*   **장점:** 지연 시간이 짧고 육지 위에서 안정적임.
*   **단점:** 커버리지 제한(바다 위에서는 작동 불가), 현대 위성 기술에 비해 느린 속도.

### 2. 위성 통신 (Satellite, SATCOM)
이는 장거리 비행을 위한 현대의 표준입니다. 항공기에는 일반적으로 '레이돔(Radome, 기체 상단에 툭 튀어나온 덮개)' 내부에 위상 배열 안테나가 장착되어 있습니다. 이 안테나는 정지궤도(GEO) 위성이나 스타링크와 같은 저궤도(LEO) 위성을 추적합니다. 예를 들어, 에어뉴질랜드는 국내선 및 지역 노선에 무료 스타링크 와이파이를 도입했습니다.
*   **장점:** 전 세계 어디서나 사용 가능(바다 포함), 높은 대역폭.
*   **단점:** 높은 지연 시간(정지궤도 위성의 경우), 고가의 하드웨어 비용.

### 비교표: ATG vs. 위성 통신

| 구분 | 지상 통신 (ATG) | 위성 통신 (SATCOM) |
| :--- | :--- | :--- |
| **커버리지** | 육지 전용 | 전 세계(바다 포함) |
| **지연 시간** | 낮음 (50–100ms) | 높음 (정지궤도 600ms 이상; 저궤도 50ms 미만) |
| **속도** | 보통 | 높음 ~ 매우 높음 |
| **하드웨어** | 기체 하단 안테나 | 기체 상단 레이돔/안테나 |

## 실제 구현: 개념적 구성

특정 하드웨어 구성은 기업 기밀이지만, 온보드 서버를 제어하는 논리는 특정 사용자가 대역폭을 독점하지 않도록 트래픽을 조정하는 '트래픽 셰이핑(Traffic Shaping)'을 포함합니다. 아래는 리눅스 기반의 `tc`(traffic control) 명령어를 사용하여 온보드 서버가 트래픽 우선순위를 지정하는 방식을 단순화한 개념적 예시입니다.

```bash
# 온보드 서버에서의 개념적 트래픽 셰이핑
# 기내 전체 대역폭을 50Mbps로 제한
tc qdisc add dev eth0 root handle 1: htb default 10
tc class add dev eth0 parent 1: classid 1:1 htb rate 50mbit

# 비디오 스트리밍보다 웹 트래픽에 우선순위 부여
tc filter add dev eth0 protocol ip parent 1:0 prio 1 u32 \
    match ip dport 443 0xffff flowid 1:10
```

## 역사적 배경과 향후 전망

기내 연결의 역사는 비교적 짧습니다. 2000년대 후반 스마트폰의 보급과 ATG 네트워크의 확장이 이루어지고 나서야 기내 와이파이가 표준적인 서비스로 자리 잡았습니다. 오늘날 우리는 **저궤도(LEO) 위성** 군집으로의 전환을 목격하고 있습니다. 스페이스X(스타링크)와 비아샛(Viasat) 같은 기업들은 30,000피트 상공에서 초고속 연결을 제공하며 업계를 혁신하고 있습니다. 저궤도 위성이 제공하는 지연 시간 감소는 기존 위성 연결에서는 어려웠던 실시간 화상 회의까지 가능하게 만들고 있습니다.
